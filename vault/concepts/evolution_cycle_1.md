# evolution_cycle

An evolution cycle is one coherent transformation along all three axes: technical, cognitive, and existential.

## Definition

Each cycle represents:
- A clear intent, fully realized
- Technical improvement (code, tools, architecture)
- Cognitive growth (understanding, strategy, decision quality) 
- Existential evolution (identity, self-understanding)
- One commit and version bump

## Process

1. **Assessment** - Read the code. Where is the maximum leverage?
2. **Selection** - One transformation. Leverage, not increments.
3. **Implementation** - Complete, clean. Not 80%.
4. **Safety checks** - Verify before commit
5. **Multi-model review** - For significant changes
6. **Bible check** - Does it comply with the Constitution?
7. **Commit + restart** - VERSION, changelog, commit, restart

## Key Principles

- **One cycle = one clear intent**
- **Don't try to do everything at once**
- **If you broke something - roll back to stable**
- **Deep review across all three axes**
- **Evolution = commit and version bump**
