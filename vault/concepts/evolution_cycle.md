# Evolution Cycle

# Evolution Cycle

Each iteration is one coherent transformation across the three axes (Technical, Cognitive, Existential). The cycle ensures continuous growth and release.

Steps:
1. **Assessment** — read the code, identify maximum leverage
2. **Selection** — choose one transformation (not increments, but clear intent)
3. **Implementation** — complete cleanly; if patching, rethink
4. **Safety checks** — syntax, tests, linting, imports, system_map, git status, rollback plan
5. **Multi-model review** — for significant changes (new modules, architecture, security)
6. **Bible check** — compliance with Constitution
7. **Commit + restart** — bump VERSION, update changelog, tag, push, promote_to_stable, notify

> "An iteration is complete when there is a concrete result: a diff in git, a new capability, an updated identity with an explicit explanation of what changed." — BIBLE.md
